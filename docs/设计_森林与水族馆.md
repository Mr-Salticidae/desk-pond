# 设计：森林与水族馆（长期反馈子版块）

> 目标：让每次使用都留下**可见的、累积的痕迹**，几周后这方世界明显更丰富。
> 状态：设计已定稿，分三期实现。第一期 = 森林。

## 调研结论（成功案例）

| 产品 | 关键做法 | 采纳 |
|---|---|---|
| Finch（自护宠物，$30M ARR） | 生命阶段成长、永不封顶；**漏签从不惩罚，宠物只是等你回来** | ✅ 只增不减，**绝不做会清零的 streak** |
| Stardew 鱼缸 / 社区中心 | 收集满 → 空间**质变** + 解锁新机制；鱼缸可**自行摆放** | ✅ 里程碑触发质变；给玩家**布置权** |
| Forest | 累积成多样化森林、按里程碑解锁新树种 | ✅ 永不封顶 + 树种多样化；❌ 不抄「分心树就死」的惩罚 |

四条原则：**永不封顶 · 非惩罚式累积 · 里程碑触发质变 · 给布置权**。

## 架构：主场景 = 可切换「房间」

3 个房间共用同一块场景画布（顶栏切换），底部番茄钟 + 任务面板**常驻**（喂养所有房间的控制台）：

```
┌ 顶栏：🏠池塘  🌲森林  🐟水族馆 ───── 图鉴 ? 置顶 — × ┐
│            [ 可切换场景区 640×210 ]                    │
├────────────────────────────────────────────────────────┤
│   番茄钟（控制台）    │      今日任务（控制台）          │
└────────────────────────────────────────────────────────┘
```

- **池塘** = "现在"（工作时看的实时陪伴，现有画面）。
- **森林 / 水族馆** = "积累"（想欣赏成果时切过去）。
- 每个房间独占整条场景带，不挤——这是「分子版块、不堆主页」的落地。

## 森林（由"任务完成"驱动）

- **拆掉 5 点封顶**（原 `tree_manager._calculate_stage`）。
- 由累计量 `tree_growth_points`（本就只增不减）**确定性推导**整片林子，第一期不新增存档字段：
  - `POINTS_PER_TREE` 点成熟一棵 → `mature = points / POINTS_PER_TREE`，当前苗 `stage = points % POINTS_PER_TREE`。
  - 第 i 棵树的**树种**按 index 阈值决定（越多越多样：阔叶 → 针叶 → 花树 → 果树 → 红叶）。
- **里程碑质变**：每解锁一个新树种，林子多一种姿态；更高里程碑点亮场景元素（小径 / 长椅 / 夜萤，留待打磨）。
- **非惩罚**：哪天没用，树不枯，只是没长新的。

## 水族馆（由"专注钓鱼"驱动，第二期）

- 钓到的鱼**真的在缸里游**，随累计变热闹；现有"图鉴"降级为缸内"目录"标签。
- 扩鱼种 + 里程碑鱼（累计达标 / 特定条件才出现），给远期钩子。
- **布置权**：挑选展示的鱼、摆放里程碑解锁的装饰（水草 / 珊瑚 / 沉船）。

## 成长账本（第三期）

只增不减的累计量：累计专注次数、累计任务、各鱼首次钓获日、**活跃天数（计数，非连胜）**。
驱动里程碑解锁与轻量「成长档案」——情感回报来源。**明确不做断签清零。**

## 数据模型

- `save_manager._merge_defaults` 已做递归默认值回填 → 新增字段对旧档自动兼容。
- 第一期：复用 `tree_growth_points`，**无需改存档**。
- 第二期起：加 `total_focus_sessions` / `aquarium_decor` / `first_caught` 等。
- 活跃天数：在 `check_new_day` 检测到换天时累加。

## 分期

1. **第一期 · 森林**：拆封顶 → 树木累积 → 树种解锁 → 森林房间 + 顶栏房间切换（落地"房间"架构）。
2. **第二期 · 水族馆**：活鱼缸房间 → 游动的鱼 → 扩鱼种 → 目录标签。
3. **第三期 · 布置与账本**：装饰摆放、成长档案、里程碑打磨。

布局约束：640×520 较挤，森林 / 水族馆超出场景带时用**轻微横向平移**，不强行缩。
